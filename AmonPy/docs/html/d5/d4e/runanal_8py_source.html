2 # @package runanal
3 # This is a top level task that gets initialized
4 # at the time that AMON server starts running.
5 # It receives event stream number, event number and event revision
6 # from the AMON server in real-time (AMON server calls task function
7 # AnalRT.run every time a new events comes from the outside world)
8 # Upon reception of these three parameters, a new event is read from the DB
9 # and passed to the main analysis code using Python multiprocessing module.
10 #
11 # Works with the Enthought Canopy package management, RabittMQ server as a broker
12 # for messages and Celery allowing from transport of messages from AMON server to
13 # this worker program.
14 #
15 # To run (outside this directory called analizer):
16 #
17 # celery worker --app=analyser --concurrency=1 -l info
18 #
19 # where dbaccess.txt contans a string in dictionary format,
20 # containing the information required to access the database
21 #
22 # Each alert will be saved to DB, and also written to XML file (in VOEvent format)
